# Этап 7. Пайплайн переобучения и автоматизация

В проект добавлен **Airflow** (папка `airflow/`) и DAG `retrain_pd_model`, который:
- по расписанию генерирует «текущие данные» (симуляция прод-потока),
- запускает Evidently отчёт,
- **триггерит переобучение**, если обнаружен дрифт (или включён `FORCE_RETRAIN=1`),
- выполняет простой quality gate по ROC-AUC.

Подробности и команды запуска — в `airflow/README.md`.
